## Special Review Notes

1. **Non-neurologic SOFA missing components:** CNS/GCS is explicitly excluded. The audit reported complete-case counts and a candidate zero-imputed score reflecting the official SOFA convention of treating missing components as 0 in the total. This is not yet a SAP decision; the formal analysis dataset must not assume ?not measured = normal/0? unless SAP explicitly approves it.

2. **Ventilation:** Formal ventilation feasibility used the MIT-LCP `ventilation` concept built from `ventilator_setting` and `oxygen_delivery`. Procedureevents and airway-procedure proxy variables were not used as the formal ventilation definition.

3. **Vasoactive dose units and overlap:** Agent-specific official SQL standardizes rates as implemented by MIT-LCP. The first-24h audit used interval overlap with ICU admission: `starttime < intime + 24 hours` and `coalesce(endtime, starttime) >= intime`. Norepinephrine-equivalent dose was built from official `vasoactive_agent` intervals.

4. **RRT chronic dialysis risk:** The official RRT concept detects dialysis/RRT-related ICU chart, input, and procedure signals. It may capture chronic dialysis delivered during ICU care as RRT exposure; distinguishing chronic maintenance dialysis from acute ICU RRT requires SAP/clinical rule review before formal use.

5. **Readmission administrative follow-up:** MIMIC patient timelines are independently date-shifted and `anchor_year_group` provides only an approximate real-year interval. Therefore exact patient-level 90-day or 365-day administrative follow-up completeness cannot be identified. The review package marks `exact_followup_complete_flag_not_identifiable = true` and proposes prespecified broad and conservative strategies.

6. **Adapted scripts on disk:** Official SQL copies are in `official_sql`; adapted SQL copies are in `duckdb_adapted`. Project-specific non-neurologic SOFA aggregate logic is stored in `run_technical_feasibility_audit.py`, whose SHA256 is recorded above.
